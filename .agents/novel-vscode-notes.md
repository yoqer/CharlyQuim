# novel-vscode Integration Notes

## Overview
- Extension registers `CustomTextEditorProvider` with view type `novel.editMarkdown` bound to `*.md` / `*.mdx` (`novel-vscode/package.json`).
- Activation wires command `novel.editWith` to trigger `vscode.openWith` for the active document (`novel-vscode/src/extension.ts`).

## Webview Plumbing
- `NovelEditorPanel.resolveCustomTextEditor` sets webview options, loads bundled React app, and posts initial `revive` message containing file text, URI, and theme (`novel-vscode/src/panels/NovelEditorPanel.ts`).
- Theme changes broadcast via `window.onDidChangeActiveColorTheme`; updates forwarded to webview as `theme` messages.
- Document edits from the webview are handled through `WorkspaceEdit.replace` on the entire file; a simple `isUpdating` flag prevents feedback loops when applying edits.
- Webview HTML loads `webview-ui/build/assets/index.{css,js}` generated by Vite with React/Tiptap UI.

## Frontend Editor (Tiptap/Novel)
- `webview-ui/src/App.tsx` mounts `Editor` from `novel` package (Tiptap-based) wrapped in `ThemeProvider`.
- Markdown extension (`tiptap-markdown`) handles Markdown ⇄ Tiptap, plus table extensions.
- On every update, the app posts `{ command: 'update', text: editor.storage.markdown.getMarkdown(), uri }` back to VS Code, ensuring Markdown string is the source of truth.
- Incoming `revive` messages reset editor content and local storage, preserving canonical Markdown.

## Takeaways for Plate Integration
- Use VS Code custom editor provider infrastructure to replace `.md` tabs with a Plate-powered webview while keeping underlying text document authoritative.
- Adopt the message bridge pattern: startup `revive` push, periodic `update` message carrying serialized Markdown, and theme sync; this keeps React editor and VS Code document in sync without adopting VS Code APIs directly inside the webview.
- Replace full-document writes with diff-aware updates later for performance, but whole-file replace is acceptable for MVP.
- Mirror `openWith` command so users can opt into the Plate editor (or set priority to default to replace Monaco).
